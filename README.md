    # 설명서 컨펌 프로젝트
    
    ## 개요
    
    서비스를 한 페이지로 설명해야 하는 온라인쇼핑몰 아이템 특성상 여러 회의를 거쳐
    컨펌받아야 함
    
    그만큼 중요한 요소이지만 시간이 너무 오래 걸리고, 컨펌이 까다롭다는 단점이 존재함
    
    이 프로젝트로 설명서 컨펌에 대한 시간 투자를 줄이되 정확하고 안전한 내용으로 구성할 수 있도록
    도울 수 있기를 생각함
    
    ## 기능
    
    ### 사용자 플로우
    
    ```
    사용자(마케터)가 아이템 설명이 들어있는 문서 OR 이미지 제시
    >> OCR & PDFparser로 내용 파악
    >> LLM에 주입해놓은 마케팅 법률 & 전략 지식들을 참고하여 설명서 *평가
    >> 설명서 신뢰도 점수를 내세우며 점수가 깎인 이유 & 근거 제시
    >> 마케터가 반영하여 재개발
    
    [평가 기준 ( 100점만점 )]
    - 마케팅 전략
    - 위법내용 유무
    - *믿을 수 있는 자료들로 구성되어있는지
    *문장에 대한 참/거짓 판별, 감정이 섞인 문장인지
     ( EX. 많은 사람들이 만족했습니다 << 검증 가능한 주장, 신뢰는 못함 )
    	만약 검증 불가능한 문장을 사용했으면 그 설명서는 사용 못하게 조치
    	(점수를 많이 깎는다, 아예 컨펌 버튼같은 걸 막아놓는다 이런 조치)
    ```
